# Аффиксы/суффиксы для Акта 1 (текущие данные)

Подождите, дайте проверю: генератор предметов в акте 1 берёт **tier_cap = act*2 = 2** и уровень предмета в диапазоне **6..10** (`ItemService.generate_inventory_item`), а кандидаты аффиксов фильтруются по:
- `Affix.tier <= tier_cap`  
- `Affix.min_level <= item_level`  
- `applies_to` (теги: `any`, `slot_type`, `attack_type`, `weapon_type`)

Источник данных: `scripts/data/affixes.json` (сидится через `scripts/seed_equipment.py`).

## Итог по акту 1

- **Всего кандидатов**: **6**  
- **Префиксы (kind=affix)**: **6**  
- **Суффиксы (kind=suffix)**: **0**  

О, я упустил в прошлых рассуждениях: в акте 1 “суффиксов” действительно нет — поэтому в магазине вы видите в основном “Мощный/Быстрый/…” без окончания после базового имени.

## Примечания “но что если…”

- **Но что если** у игрока в акте 1 выпадает предмет уровня ниже 6? В текущем генераторе для акта 1 уровень предмета считается как `tier_cap*5 - 4 + rand(0..4)` → это всегда `6..10`, поэтому min_level=6 для всех базовых аффиксов совпадает.
- Если хотите разнообразия “прямо в акте 1”, нужно расширять `affixes.json`:
  - добавить tier=1 аффиксы с `min_level=1..5`,
  - добавить `kind="suffix"` варианты,
  - и/или добавить аффиксы с `applies_to` для конкретных `slot_type` (например, только `weapon_1h`, только `ring` и т.д.).
