## 1. Архитектура сложностей

### Слоты экспедиций — свободный выбор
Вместо 3 фиксированных слотов «лёгкий/средний/тяжёлый» — игрок сам конфигурирует:
- Выбирает **тип аффикса** (Гоблины, Пауки, Нежить и т.д.)
- Выбирает **уровень аффикса** (I–V)
- Выбирает **длительность** (30/45/60/90/120 мин)
- Выбирает **состав отряда** (1-3 наёмницы)

Никакого «уровня экспедиции привязанного к ОВ» — сложность определяется только уровнем аффикса и длительностью.

### Уровни аффиксов (I–V)
```
Аффикс I   → базовый HP урон за событие: 6% HP отряда
Аффикс II  → 10%
Аффикс III → 15%
Аффикс IV  → 20%
Аффикс V   → 28%
```

Игрок не видит % успеха заранее — только видит состав отряда и аффикс. ИИ делает остальное.

## 2. Формула урона за событие

```python
def calc_event_damage(
    base_hp_pct: float,        # от уровня аффикса (0.06–0.28)
    squad_hp_total: int,       # суммарное макс HP отряда
    race_counters: int,        # сколько наёмниц с расовым бонусом (каждая -8%)
    class_counters: int,       # сколько наёмниц с классовым бонусом (каждая -15%)
    perk_level: int,           # уровень перка-контра (0 = нет)
    difficulty_level: int,     # уровень аффикса (1-5)
    rand_variance: float,      # rand(0.85, 1.15)
) -> int:
    mult = 1.0
    
    # Расовый бонус: -8% за каждую наёмницу с расовым бонусом (макс -24% для отряда 3)
    mult *= (1 - 0.08) ** race_counters
    
    # Классовый бонус: -15% за каждую наёмницу с классовым бонусом
    mult *= (1 - 0.15) ** class_counters
    
    # Перковый бонус: до -35%, пропорционально разнице уровней
    if perk_level > 0:
        effectiveness = min(1.0, perk_level / difficulty_level)
        mult *= (1 - 0.35 * effectiveness)
    
    # Дисперсия ±15%
    mult *= rand_variance
    
    base_damage = squad_hp_total * base_hp_pct
    return max(1, round(base_damage * mult))
```

### Примеры для отряда 300 HP:
| Аффикс | Отряд | Урон без контра | Урон с расой | Урон с классом | Урон с перком(3/3) |
|---|---|---|---|---|---|
| Гоблины I  | 300 HP | 18 HP (6%) | 16 HP | 15 HP | 12 HP |
| Гоблины III | 300 HP | 45 HP (15%) | 41 HP | 38 HP | 29 HP |
| Гоблины V  | 300 HP | 84 HP (28%) | 77 HP | 71 HP | 65 HP |
| Гоблины V, перк 1/5 | 300 HP | 84 HP | — | — | 77 HP (перк слабый) |

### Перки → конкретные категории (-35% при полном контре, снижается при разнице уровней)
60 перков делятся на 7 категорий:
- enemy: 22 перка (COMBAT × 5, STEALTH × 2, DEFENSE × 3, ...)
- hazard: 20 перков (TRAP × 5, HEALING × 3, NATURE × 2, ...)
- social: 16 перков (SOCIAL × 5, TRADE × 5, KNOWLEDGE × 2, ...)
- cursed: 13 перков (SPIRIT × 5, MAGIC × 3, HEALING × 2, ...)
- nature: 7 перков (NATURE × 5, KNOWLEDGE × 1, ...)
- knowledge: 9 перков (KNOWLEDGE × 5, LUCK × 2, ...)
- magic: 6 перков (MAGIC × 5, SPIRIT × 1)

## 4. Система пула наёмниц (замена отряда)

**Отказ от системы «отряд/запас»** → единый пул до 10 наёмниц.

Правила:
- Наёмница **в экспедиции** — заблокирована: нельзя отправить в другую, нельзя продать, нельзя нанять замену сверх лимита 10
- Наёмница **раненая** (HP < 25%) — нельзя отправить в новую экспедицию
- Максимум **3 параллельных экспедиции** одновременно
- Лимит пула: 10 наёмниц (не зависит от уровня ОВ)

БД:
```sql
-- Поле expedition_id в hire_units уже есть

  CHECK (expedition_id IS NULL OR hp_current >= 1);
```

## 7. Выбор испытания с учётом состава отряда

При генерации каждого события — выбирать испытание с повышенным весом для категорий где у отряда есть перки:

```python
def weighted_challenge_selection(squad, available_challenges):
    # Собрать все категории перков отряда
    squad_categories = set()
    for unit in squad:
        for perk in unit.perks:
            squad_categories.update(PERK_COUNTERS[perk.id])
    
    weights = []
    for challenge in available_challenges:
        base_weight = challenge.weight
        # Если категория испытания есть в перках отряда — повысить вес ×2
        if challenge.category in squad_categories:
            base_weight *= 2
        weights.append(base_weight)
    
    return random.choices(available_challenges, weights=weights)[0]
```

Это гарантирует что перки «используются» чаще — без жёсткой привязки.
