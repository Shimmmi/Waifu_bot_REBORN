# UX/UI вкладки «Экспедиции» v2.0 — дизайн-документ

**Версия:** 2.0
**Статус:** UX/UI Redesign (с поправками на актуальные механики v2)
**Цель:** привести интерфейс экспедиций к стандартам мобильных RPG/idle-игр, снизить когнитивную нагрузку и ускорить основной сценарий «Отправить экспедицию».

---

## Актуальные механики v2 (краткая справка)

| Параметр | Значение |
|----------|----------|
| Глубины I–V | Разведка(30м,2тик) / Патруль(45м,3) / Поход(60м,4) / Рейд(90м,6) / В глубину(120м,8). Тик = 15 мин |
| Мощность | `RARITY_BASE + (level-1)×3` (Common 40 → Legendary 120). Минимум по тиру: 0/80/150/220/300 |
| Исход | по остатку HP отряда после всех тиков: `<12% → Провал`, `≥52% → Успех`, иначе `Частичный`. Предroll-шанса НЕТ |
| Типы наград (6) | `gold`, `waifu_exp`, `items`, `enchant`, `merc_exp`, `mixed` |
| Параллельных слотов | 3 (`EXPEDITION_MAX_CONCURRENT`) |
| Состав | 1–3 наёмника, пул до 10, HP≥25% для участия |
| Теги/аффиксы | генерируются СЛУЧАЙНО на старте; при формировании отряда НЕ известны |
| Лечение | после экспедиции, `0.8 мин × % потерянного HP` (×1.5 если HP=0) |
| Отмена | 50% наград (`EXPEDITION_CANCEL_REWARD_PCT`) |
| Твисты | ~10%: treasure / rest / skip damage и т.п. |

## A.1 Главный экран «Экспедиции»

Иерархия сверху вниз:

```
Верхняя панель (level + XP, золото, чип данжа)
↓
Вкладки (⚔ Одиночные / 🗺 Экспедиции / 👥 Групповые / 🕳 Бездна)
↓
Активные экспедиции (до 3 карточек) + индикатор слотов «2/3 занято»
↓
Большая кнопка «Отправить экспедицию» (primary CTA)
↓
Свободные слоты / мини-таблица тиров I–V
↓
Нижняя навигация
```

---

## A.2 Верхняя панель

- **Уровень основной вайфу** + компактный XP-бар с подписью «Уровень».
- **Золото** с подписью «Золото».
- **Чип активного данжа — оставить** (информативен). Чип экспедиций ранее удалён — не возвращать.
- Чип данжа визуально подчинён primary CTA (меньший вес, нейтральный цвет), не конкурирует с ним.
- Никакой псевдо-«Главы 4-4» для экспедиций не существует.

---

## A.3 Вкладки навигации

- Тематические иконки: `⚔ Одиночные` · `🗺 Экспедиции` · `👥 Групповые` · `🕳 Бездна`.
- Активная вкладка: более яркий фон, золотое свечение, жирный текст, плавный transition.
- **Компактность**: шрифт ~11px, уменьшенные горизонтальные отступы, `white-space:nowrap`, `overflow:hidden`, `text-overflow:ellipsis` — чтобы 4 длинных названия с иконками помещались в одну строку без переносов.

## A.4 Карточка активной экспедиции

```
[Биом-арт / градиентный fallback]
[Архetype chip + иконки сгенерированных аффиксов (до 4)]
Название локации (AI-сгенерированное)
Тир: «Рейд (IV)» · длительность «90 мин»
Статус: ● В пути / ● Готово
Прогресс: events_done/events_total + полоса + обратный отсчёт
Сила отряда: ⚔ 224 / рекоменд. 220
HP отряда: ❤ 78%
Награда (тип + превью): 💰 120 золота · ⚗ 2 камня
```

- Статусы строго: `🟡 В пути`, `🟢 Готово`, `⚫ Отменена`. Никаких «Возвращается»/«Провал» как активного статуса.
- Таймер: «Осталось 02:45:17» для активных; «Готово к получению» для завершённых.
- На «Готово»: пульсация статуса + мигание CTA «Забрать».
- Цветовая рамка по числу сложностей (`exp-diff-count-1..5`).

---

## A.5 Кнопка «Отправить экспедицию» — primary CTA

- Ширина почти во весь контент-блок, увеличенная высота, насыщенный золотой фон, мягкое свечение.
- Sticky-закрепление над нижней навигацией при прокрутке.
- Disabled-состояние с подсказкой, если 3/3 слотов заняты («Все слоты заняты»).

---

## A.6 Нижняя часть экрана

- Если есть свободные слоты (active < 3): плейсхолдеры «Свободно» с призывом «Отправить экспедицию».
- Если 3/3 занято: краткая лента «Последняя добыча» (1–2 последние клейма с outcome-иконкой).
- Раскрывающаяся по тапу мини-таблица тиров I–V (длительность + рекоменд. мощность + урон/тик).

---

## A.7 Модальное окно отправки

Последовательный поток сверху вниз:

```
Заголовок: «Новая экспедиция»
↓
Мощность отряда: ⚔ 224 / рекоменд. 220 (цвет по порогу)
Прогноз исхода по HP: «Урон ~15% за тик × 4 (базовый для тира III)»
  → «Ожид. остаток HP ~40% без учёта случайных факторов»
  → подпись «приблизительный прогноз; реальные сложности и твисты определятся случайно после старта»
  (теги/аффиксы сложности НЕ показываются — они генерируются случайно на старте)
↓
Тип награды (6 плиток):
  💰 Золото · ⭐ Опыт ОВ · 🗡 Снаряжение · 💎 Камни заточки · ✨ Опыт наёмниц · 🎁 Смешанная
↓
Глубина (I–V) — плитки с реальными данными:
  I  Разведка    30 мин · 2 события · рекоменд. 0   · урон 6%/тик
  II Патруль     45 мин · 3 события · рекоменд. 80  · урон 10%/тик
  III Поход      60 мин · 4 события · рекоменд. 150 · урон 15%/тик
  IV Рейд        90 мин · 6 событий · рекоменд. 220 · урон 20%/тик
  V  В глубину  120 мин · 8 событий · рекоменд. 300 · урон 28%/тик
  Заблокировано, если squad_power < min.
↓
Состав отряда (1–3 карточки-слота)
↓
Список наёмников (с фильтрами и поиском)
↓
Нижняя панель подтверждения:
  ⚔ Мощность 224 · ❤ Прогноз HP ~40% · ⏱ 60 мин · 💰 120 золота
  [ОТПРАВИТЬ ЭКСПЕДИЦИЮ]
```

Ключевые отличия от v1.0:
- **«Шанс успеха %» заменён** на «Прогноз исхода по HP» (по базовому урону тира × число событий, без variance/твистов).
- **Теги/аффиксы сложности НЕ показываются на этапе формирования** — они генерируются случайно только после старта. Поле `tag_preparation_pct`/`tag_effectiveness_pct` из `/expeditions/preview` в UI отправки НЕ отображать. Влияние тегов раскрывается постфактум в логе результата (A.9).
- **6 типов наград** (без выдуманных «самоцветов»).
- **Тиры с реальными длительностями 30–120 мин** и порогами мощности 0–300.

## A.8 Карточка наёмника в списке

```
[Портрет]
Имя · LV49
Редкость · Роль
❤ 200/200
Статус-плашка (если занят):
  «🔒 В экспедиции» / «💊 Лечение до 12:35» / «❤ Мало HP» (полупрозрачная серая плашка)
```

Фильтры (актуальные состояния): **Все / Готовы / В экспедиции / Лечение**. Плашка «В бою» убрана — не применима к наёмникам.

Сортировка: по уровню / по силе / по редкости / по роли. Поиск по имени.

## A.9 Экран результата экспедиции

- Исход крупным цветным баннером: `✅ Успех` / `⚠ Частичный успех` / `✖ Провал`.
- AI-нарратив (короткий абзац).
- **Лог ворот (gate log) с расшифровкой влияния сложностей на математику** — ключевой источник информации, поскольку на этапе формирования теги были скрыты:
  - Список выпавших тегов/аффиксов данной экспедиции (с иконками и названиями).
  - Для каждого тика: базовый урон тира, множитель покрытия тегами (race/class/perk), категория вызова (±10%), variance-ролл, итоговый урон по отряду.
  - Твисты с пометкой типа (`treasure` / `rest` / `skip damage` / …) и их численный эффект.
  - Итоговый HP-остаток отряда → какой порог исхода сработал (`<12% → Провал`, `≥52% → Успех`, иначе `Частичный`).
- Полученные награды с разбивкой по типам (с множителем outcome: ×1.0 / ×0.7 / ×0.4).
- Статус наёмников: кто требует лечение и на сколько (прогноз `0.8 мин × % потерянного HP`).
- CTA «Забрать» (забирает награды, запускает лечение, закрывает модалку).

## A.10 Цветовая система

| Цвет | Назначение |
|------|-----------|
| Зелёный | Готово / Успех |
| Жёлтый | В пути / Частичный успех |
| Красный | Провал / Отмена / Недостаточно мощности |
| Синий | Информация / лечение |
| Фиолетовый | Эпические награды / камни заточки |
| Золотой | Primary CTA / активная вкладка |

## A.11 Типографика

H1 24–28px (заголовки экранов/модалок) · H2 18–20px (названия карточек) · Body 14–16px · Caption 11–12px.

---

## A.12 Микроанимации

- Выбор героя: увеличение карточки + золотая рамка + плавное появление в составе.
- Выбор тира: подсветка + масштабирование + smooth transition.
- Завершение экспедиции: пульсация статуса + мигание «Забрать».
- Лечение запущено: плавное появление таймера на карточке наёмника.
